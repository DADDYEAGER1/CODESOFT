## CONTENT APPROACH

**Types of claims allowed:**
- Objective performance comparisons with data
- Cost analyses with specific numbers
- Trade-off explanations for each option
- Priority-based matching (best for X if you value Y)

**Types of claims forbidden:**
- Universal "better" claims without context
- Harsh dismissal of any valid option
- Subtle bias through unequal treatment

**Evidence standards:**

Every comparison requires:
1. **Clear criteria** (what you're evaluating and why)
2. **Equal treatment** (same depth of analysis for each option)
3. **Specific data** (costs, timeframes, performance metrics)
4. **Balanced assessment** (strengths AND limitations for each)
5. **Priority matching** (who each option suits best)
6. **Trade-off clarity** (what you gain and sacrifice with each)

**Critical balance check:** Count sentences for each option. If Option A gets 8 sentences and Option B gets 3, you're not balanced. Equal word count is a proxy for fair treatment.

**Formatting preferences:**
- Use parallel structure for side-by-side comparison
- Create comparison tables for 3+ criteria
- Use subheadings that treat options equally
- Include "Choose X if..." / "Choose Y if..." sections
- Bold key differentiators for scannability
- Present strengths/limitations in balanced lists

**Structure guidelines:**
- Open with what's being compared and why it matters
- Establish comparison criteria explicitly
- Present Option A with full analysis (strengths, limitations, costs, performance)
- Present Option B with equally thorough analysis (using parallel structure)
- Provide direct trade-off comparison
- Create decision framework based on priorities
- Close acknowledging context-dependent superiority
